# coding-rules.md — 编码规则约束

## 1. 语言标准

- 使用 **C99** 标准，禁止 C++ 特性（除非明确标注 C++ 模块）
- 文件编码：UTF-8（无 BOM）
- 行结束符：LF（Unix 风格）

## 2. MISRA-C:2012 规则

| 规则编号 | 描述 | 级别 |
|---|---|---|
| Rule 1.3 | 禁止未定义/未指定行为 | 必须 |
| Rule 2.1 | 不可达代码必须删除 | 必须 |
| Rule 8.4 | 外部链接函数/变量必须有声明 | 必须 |
| Rule 11.3 | 禁止对象指针与非对象指针互转 | 必须 |
| Rule 14.3 | 控制表达式不得为不变量 | 必须 |
| Rule 15.5 | 函数只有一个返回点（建议） | 建议 |
| Rule 17.7 | 非 void 函数返回值不得忽略 | 必须 |

## 3. 命名规范

```
模块名_功能描述()                   // 函数：Pascal风格带模块前缀
MODULE_CONSTANT_NAME               // 宏/常量：全大写下划线
ModuleType_VariableName            // 类型定义：Pascal风格
u8LocalVariable / u16GlobalVar     // 变量：匈牙利记法（可选）
```

## 4. 内存使用约束

- ❌ 禁止 `malloc` / `free` 动态分配
- ❌ 禁止递归调用（防止栈溢出）
- ✅ 所有缓冲区静态分配，大小编译期确定
- ✅ 全局变量初始化须显式赋值

## 5. 中断服务程序（ISR）规范

- ISR 函数名格式：`MODULE_IRQHandler(void)`
- ISR 内禁止调用非可重入函数
- ISR 执行时间 ≤ 50μs（具体由系统设计文档规定）
- ISR 内禁止 `printf` 等阻塞操作

## 6. 安全关键代码规范

- ASIL-B 及以上代码必须添加 `/* @ASIL-B */` 或 `/* @ASIL-D */` 注释标注
- 关键变量必须有冗余备份或奇偶校验
- 所有外部输入必须进行范围检查

## 7. 注释规范

```c
/**
 * @brief   函数简要描述
 * @param   paramName  参数说明
 * @return  返回值说明
 * @note    注意事项（可选）
 * @asil    ASIL等级（安全关键函数必填）
 */
```
